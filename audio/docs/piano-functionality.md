# piano.html 功能梳理

`piano.html` 是一个基于 Web Audio API 实现的浏览器虚拟钢琴页面。它本身作为页面的骨架，结合了 CSS 样式表和 JavaScript 模块来实现用户界面和交互逻辑。以下是其具体功能的详细梳理：

## 1. 页面结构与样式
- **骨架结构**：页面提供了一个 `<div id="piano-keyboard" class="piano-keyboard"></div>` 容器，用于后续通过 JavaScript 动态挂载钢琴的黑白按键（DOM 元素）。
- **样式引入**：引入了外部 CSS 文件 `./style/piano.css`，用于控制钢琴键盘的视觉表现。
- **用户体验优化**：通过内联样式 `* { user-select: none; }` 禁用了页面元素的文本选中，防止用户在频繁点击和操作按键时发生误触或文本被高亮的现象，保证了弹奏时的流畅体验。

## 2. 核心逻辑 (与 js/piano.js 联动)
页面通过 `<script type="module">` 引入并实例化了 `piano.js` 中的 `Piano` 类（`window.piano = new Piano();`）。`Piano` 类的主要功能包含：

### 2.1 模拟钢琴键盘生成
- **按键生成**：程序动态创建了 3 个八度（C4 到 B6）的琴键，包括 白键 和 黑键。总计跨越音阶 4、5、6。
- **DOM 挂载**：根据设定的音名（如 C4, #C4, D4 等），将各个琴键（封装在 `WhiteKey` 和 `BlackKey` 类中）渲染为 HTML DOM 节点，并插入到页面中的 `piano-keyboard` 容器内。

### 2.2 声音播放机制
- **AudioContext 实例**：在实例化 `Piano` 时，会创建一个原生的 `AudioContext` 实例，这是 Web Audio API 的核心对象，用于处理和合成音频。
- **NotePlay 联动**：每个琴键被赋予一个 `NotePlay` 实例（来自 `./music/NotePlay.js`），负责根据对应的音高（频率）和演奏指令来发声。

### 2.3 交互控制与事件绑定
支持 **鼠标** 和 **电脑键盘** 两种弹奏方式：
- **鼠标模拟**：监听每个按键元素的 `mousedown` 事件，在鼠标按下时触发发音（`play()` 方法）。
- **键盘映射 (键名与简谱支持)**：监听了全局的 `keydown` 和 `keyup` 事件，通过正则和映射表，将电脑键盘输入转化为特定的音高：
  - **基础音阶 (第 4 八度)**：
    - 字母键：`a, s, d, f, g, h, j` 分别对应 C4 到 B4 (即简谱 1 到 7)。
    - 数字键：小键盘的 `1` 到 `7` 同样对应 C4 到 B4。
    - 字母键（黑键）：`w, e, t, y, u` 对应 #C4, #D4, #F4, #G4, #A4。
  - **高音阶切换 (组合按键)**：
    - 借助 `Shift` 键：如 `Shift + a` (内部识别为 `a+s`)，或者 `Shift + 1`，将切换到第 5 八度（C5 到 B5）。
    - 借助 `Ctrl` 键：如 `Ctrl + a` (内部识别为 `a+c`)，或者 `Ctrl + 1`，将切换到第 6 八度（C6 到 B6）。
- **动态视觉反馈**：按键按下时（`keydown`），会为其 DOM 元素添加 `is-keydown` 类名以展示按下效果；松开时（`keyup`）移除该类名以恢复原状。

---

**总结**：`piano.html` 是一个轻量然而功能完备的网页虚拟钢琴。用户打开页面后，既可以直接使用鼠标点击屏幕上的琴键，也可以通过电脑键盘及 Alt/Ctrl 组合键，演奏出不同八度、包含黑白键的音乐。
